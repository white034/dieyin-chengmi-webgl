# 《叠印成谜》Unity 完整原型

面向微信小游戏方向验证的竖屏印章解谜原型。玩家选择印章，在画纸上拖动、旋转并盖印，通过叠色与镂空还原顶部持续显示的目标轮廓；相似度达到 90% 且不超过盖章次数即可通关。

## 已实现

- Unity `2022.3.62f3c1` 项目结构，目标平台 WebGL
- 1080×1920 竖屏自适应，鼠标与触摸共用交互
- 拖动预览、0.05 归一化网格吸附、每次 15°旋转
- CPU 像素印章合成与实时 IoU 相似度，不依赖计算着色器
- 覆盖印、镂空印、落印顺序影响结果
- 第 1 关橘子教学：2 枚正确印章，附 1 枚干扰印章
- 第 2 关舞狮极难关：13 枚必需印章、7 枚干扰印章、13 次上限
- 固定可解标准答案；标准答案会由编辑器脚本校验至 100%
- 关卡解锁、三星、无限重试、本地 PlayerPrefs 存档
- 每日谜题日期记录，仅当天首次通关计入记录
- 撤销、提示、失败/成功弹窗、模拟激励广告复活
- 程序化印章图形、程序化音效、震动、落墨粒子
- 原创国潮宣纸背景与 Noto Sans SC 中文字体

## 打开与运行

1. 用 Unity Hub 安装官方 Unity `2022.3.62f3c1`，并勾选 **WebGL Build Support**。
2. 在 Hub 中打开本目录。
3. 打开 `Assets/Scenes/Main.unity`，点击 Play。
4. 首次导入后可执行菜单 `叠印成谜 > 验证关卡`。

## 操作

- 点击印章：选中印章。
- 拖动印章到画纸：显示实时落印预览，松手即盖印。
- `左转15°` / `右转15°`：旋转当前印章。
- `撤销`：免费撤销一次；再次撤销会打开模拟激励广告。
- `提示`：高亮标准解中下一枚尚未使用的印章。
- 相似度达到 90% 后立即通关；次数用尽且未达标则失败。

## WebGL 构建

编辑器菜单：

1. `叠印成谜 > 配置项目`
2. `叠印成谜 > 验证关卡`
3. 在 Build Settings 中切换 WebGL 后构建

也可批处理调用：

```powershell
Unity.exe -batchmode -quit -projectPath "<项目目录>" -executeMethod ProjectBuilder.PerformBuild -logFile "<日志文件>"
```

成功输出位于 `Build/WebGL`，并生成 `BUILD_OK.txt`。WebGL 压缩已设为 Disabled，便于用普通静态服务器本地预览；正式发布时可按服务器配置改为 Brotli 或 Gzip。

## 代码结构

- `GameModels.cs`：关卡、印章和固定标准解数据
- `InkEngine.cs`：CPU 遮罩合成、预览、镂空与相似度计算
- `StampCard.cs`：印章卡片点击/拖动
- `BoardDragSurface.cs`：画纸拖动坐标转换与落印
- `GameBootstrap.cs`：运行时 UI、流程、存档、广告模拟和反馈
- `AudioRuntime.cs`：WebGL 兼容的程序化音效
- `ProjectBuilder.cs`：配置、关卡验证与 WebGL 自动构建

## 接入微信小游戏前

本原型没有绑定微信 SDK、真实广告 SDK 或服务端。正式接入时建议将 `PlayerPrefs` 替换为微信云存档，将模拟广告回调替换为激励视频广告，并用 Unity WebGL 转微信小游戏的官方/受支持方案完成包体转换。
